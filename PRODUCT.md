# Калькулятор торта — опис продукту

Документ описує всі функції продукту, щоб не доводилось реверс-інжинирити їх із коду.
Єдиний файл продукту: index.html — самодостатня сторінка (HTML + CSS + JS в одному файлі).

## Призначення

Інструмент для кондитера: за вже випеченим тортом, який вийшов добре, обчислюється **щільність** цього типу торта. Далі за цією щільністю калькулятор перераховує параметри нового торта (вагу, висоту, діаметр або сторону) так, щоб тісто/начинка лишались такої ж консистенції.

Кожен торт (референсний і новий) може бути **круглим** (циліндр) або **прямокутним** (коробка). Форми обираються незалежно, тож можна перераховувати в будь-якій комбінації: круглий → круглий, круглий → прямокутний, прямокутний → круглий, прямокутний → прямокутний.

Основна ідея: щільність — це вага / обʼєм, і вона не залежить від форми. Щільність референсу переноситься на новий торт, бо це той самий рецепт.

## Технічна основа

- Одна HTML-сторінка, мова інтерфейсу — українська (`lang="uk"`), числа форматуються через `toLocaleString("uk-UA")`.
- 3D-візуалізація на **Three.js 0.160.0**, що підвантажується з CDN (`jsdelivr`) через `importmap`. Потрібен інтернет для роботи 3D-сцени.
  - Модулі: `three`, `OrbitControls`, `CSS2DRenderer`/`CSS2DObject`.
  - Геометрія залежить від форми: `CylinderGeometry` для круглого торта, `BoxGeometry` для прямокутного.
- Без збірки, без залежностей у репозиторії — відкривається подвійним кліком у браузері.
- Адаптивна верстка: на ширині ≤ 800px колонки складаються в одну (`@media`).
- Тема оформлення задається CSS-змінними в `:root` (бежево-коричнева палітра, акцент `--accent: #8b5e34`).

## Структура екрана

Три блоки + розкривний блок формул.

### Блок 1. Референсний торт

Торт, який уже робили. Спочатку обирається **форма** перемикачем `Кругла` / `Прямокутна` (за замовчуванням — кругла). Залежно від форми показуються різні поля розмірів:

- **Кругла:** Діаметр (см, за замовч. `16`).
- **Прямокутна:** Довжина (см, за замовч. `20`) і Ширина (см, за замовч. `20`).
- Спільні для обох форм: Висота (см, за замовч. `10`) і Вага (г, за замовч. `1300`).

Результат блоку:
- **Щільність** цього типу торта в г/см³ (3 знаки після коми).
- Пояснення «100 см³ важать приблизно X г» (щільність × 100).

### Блок 2. Новий торт

Так само спочатку обирається **форма** (`Кругла` / `Прямокутна`).

Перемикач режиму (що саме рахувати) **залежить від форми**:
- **Кругла:** `Вагу` · `Висоту` (активна за замовч.) · `Діаметр`.
- **Прямокутна:** `Вагу` · `Висоту` · `Довжину` · `Ширину`.

Поведінка режиму:
- Поле обраного параметра стає **тільки для читання** (клас `readonly`, `readOnly = true`) і заповнюється обчисленим значенням.
- Інші поля вводяться вручну.

Зміна форми коригує режим, якщо він став недопустимим (`coerceMode`):
- при переході на круглу: `довжина`/`ширина` → `діаметр`;
- при переході на прямокутну: `діаметр` → `довжина`;
- `вага`/`висота` зберігаються завжди.

Поля нового торта (значення за замовчуванням):
- Кругла: Діаметр — `22`.
- Прямокутна: Довжина — `30`, Ширина — `20`.
- Спільні: Висота — порожньо (бо стартовий режим — рахувати висоту), Вага — `5000`.

Результат блоку:
- Велике значення обчисленого параметра з одиницею (г / см).
- Текстове пояснення з підставленими числами (вага показується ще й у кг; для прямокутної форми основа показується як `довжина × ширина`).
- **Статус поточної щільності нового торта** — порівняння фактичної щільності нового торта з референсом:
  - Показує щільність нового торта (г/см³) і різницю у відсотках.
  - Колір/стан статусу: **зелений (ok)**, якщо різниця ≤ 3% за модулем; інакше **червоний (warn)**.

### Блок 3. Порівняння старого і нового торта (3D)

Інтерактивна 3D-сцена, де два торти малюються **в одному масштабі** поруч. Круглий торт — циліндр, прямокутний — коробка (форми можуть бути різними одночасно).
- Референсний торт — фіолетовий (`#9f7aea`), новий — коричневий (`#8b5e34`). Є легенда з кольорами.
- Масштаб підбирається автоматично: `unit = 8 / max(maxHoriz, maxH)`, де `maxHoriz` — найбільший горизонтальний розмір серед обох тортів (для круглого це діаметр, для прямокутного — довжина і ширина), а `maxH` — найбільша висота.
- Розташування: фронтальний розмір (діаметр або довжина) йде вздовж осі X, глибина (діаметр або ширина) — вздовж Z; торти розставлені з проміжком (`gap = 1.2`), мінімальна висота — `0.05`, щоб тонкий торт лишався видимим.
- Камера автоматично кадрує сцену (`frameScene`), щоб обидва торти повністю вміщувались (з урахуванням і ширини, і глибини).

**Керування сценою:**
- Перетягування мишею — обертання (OrbitControls, з демпфуванням).
- Колесо миші — масштаб (зум обмежений: `minDistance = 4`, `maxDistance = 60`).
- Кнопка **«Скинути вигляд»** — повертає камеру в дефолтний ракурс.
- Кнопка **«Підказки»** (активна за замовчуванням) — вмикає/вимикає текстові підписи на тортах.

**Підписи на кожному торті (CSS2D-мітки):**
- Заголовок («Референс» / «Новий») — завжди видимий.
- Вага в кг — перемикається кнопкою «Підказки».
- Розмір основи — перемикається: для круглого `d X см`, для прямокутного `довжина × ширина см`.
- Висота `h X см` — перемикається.

**Текстові примітки під сценою:**
- Щільність референсу в г/см³.
- Рядок порівняння: у скільки разів новий торт більший за основою, висотою, вагою; пропорції основи до висоти обох тортів.

**Три картки співвідношень (ratio-grid):** пропорція рахується як фронтальний розмір / висота (для круглого це діаметр, для прямокутного — довжина). Заголовки й пояснення карток підлаштовуються під форму (`діаметр` або `довжина`).
1. **Референс: діаметр/довжина / висота** — у форматі `X : 1` + пояснення «на 1 см висоти припадає X см діаметра/довжини».
2. **Новий: діаметр/довжина / висота** — те саме для нового торта.
3. **Різниця пропорції** — у відсотках, з текстовим висновком:
   - різниця ≤ 5% → «форма буде схожа»;
   - > 0 → «новий торт буде ширший і нижчий відносно висоти»;
   - < 0 → «новий торт буде вищий і вужчий».

### Блок формул (розкривний `<details>`)

Показує використані формули для обох форм:
- Обʼєм круглого (циліндр): `V = π × (діаметр / 2)² × висота`
- Обʼєм прямокутного (коробка): `V = довжина × ширина × висота`
- Щільність референсу: `ρ = вага / V`
- Перерахунок кругла: вага `m = ρ × π × (діаметр / 2)² × висота`; висота `h = m / (ρ × π × (діаметр / 2)²)`; діаметр `d = 2 × √(m / (ρ × π × висота))`
- Перерахунок прямокутна: вага `m = ρ × довжина × ширина × висота`; висота `h = m / (ρ × довжина × ширина)`; довжина `довжина = m / (ρ × ширина × висота)`; ширина `ширина = m / (ρ × довжина × висота)`

## Логіка обчислень

- **Обʼєм:** кругла — `π × (d/2)² × h`; прямокутна — `довжина × ширина × h`.
- **Щільність референсу:** `ρ = refWeight / V(референс)`.
- **Режими (кругла):**
  - вага: `m = ρ × π × (newD/2)² × newH` (округлюється до цілих грамів);
  - висота: `h = newW / (ρ × π × (newD/2)²)`;
  - діаметр: `d = 2 × √(newW / (ρ × π × newH))`.
- **Режими (прямокутна):**
  - вага: `m = ρ × newL × newW × newH` (округлюється до цілих грамів);
  - висота: `h = newWeight / (ρ × newL × newW)`;
  - довжина: `довжина = newWeight / (ρ × newW × newH)`;
  - ширина: `ширина = newWeight / (ρ × newL × newH)`.
- **Поточна щільність нового торта:** `newWeight / V(новий)`, різниця з референсом у %.

### Валідація та форматування

- `number()` приймає лише скінченні числа **> 0**; інакше повертає `null` (порожні/нульові/відʼємні значення → прочерк «—» і підказка ввести коректні дані).
- `getCakeInput()` збирає розміри торта залежно від форми; `dimsComplete()` перевіряє, що всі потрібні поля заповнені (кругла — діаметр+висота+вага; прямокутна — довжина+ширина+висота+вага).
- `round()` — локалізоване форматування; для значень < 10 показує мін. 2 знаки.
- `formatForInput()` — значення для полів вводу з фіксованою кількістю знаків (за замовч. 1).
- Перерахунок відбувається на кожну зміну будь-якого поля (подія `input`), при перемиканні режиму та при зміні форми.

## Точки розширення / відомі нюанси

- **Стан у адресному рядку:** параметри зберігаються в query-рядку URL і відновлюються при завантаженні.
  - Форма — `rs` (референс) і `ns` (новий), значення `round`/`rect`. Якщо їх немає — кругла (зворотна сумісність зі старими посиланнями).
  - Референс — `rd` (діаметр, кругла) **або** `rl`/`rb` (довжина/ширина, прямокутна), плюс `rh` (висота), `rw` (вага). Записуються параметри лише активної форми.
  - Новий торт — `mode` (`weight`/`height`/`diameter`/`length`/`width`) + `nd` **або** `nl`/`nb`, плюс `nh`/`nw`; **опціональні**. Якщо їх немає в URL, лишаються значення за замовчуванням. Це дозволяє будувати посилання ззовні лише з референсом і переадресовувати на сторінку.
  - URL оновлюється через `history.replaceState` (без засмічення історії) на кожну зміну поля, при перемиканні режиму та форми. Обчислене (readonly) поле теж потрапляє в URL уже з порахованим значенням.
  - Читання при старті: `applyParamsFromURL()` → `setShape("ref", …)`, далі `setShape("new", …)` з уже скоригованим `mode`. Невідомий `mode` ігнорується (фолбек на `height`).
  - Немає localStorage — стан живе лише в URL.
- **Залежність від CDN:** офлайн 3D-сцена не завантажиться; решта калькулятора (обчислення) теж у тому ж `<script type="module">`, тож при недоступному CDN сторінка не порахує нічого. Варто враховувати при потребі офлайн-режиму.
- **Один файл:** уся логіка, стилі й розмітка — в `index.html`.
